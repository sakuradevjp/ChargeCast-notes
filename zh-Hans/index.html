3 分钟开播 —— 快速演示

从「也许我也能直播」到「我已经在直播了」的最短路径。

1插上 USB 线

在 Android 手机上打开 USB 调试，然后用线连接到电脑。
用日常的充电线就行。 没错 —— 直播全程手机都在充电。

第一次用？启用 USB 调试（2 分钟）
  1. 打开 Android 设置 → 关于手机
  2. 连续点击 版本号 7 次，直到出现「您已进入开发者模式」
  3. 进入 设置 → 系统 → 开发者选项，打开 USB 调试
  4. 插上线后，在权限提示上点 允许

各厂商菜单路径略有不同（三星：设置 → 关于手机 → 软件信息 → 点击版本号）。

2挑个预设，点 Start Capture

一键选择 720p / 1080p / 1440p / Custom。scrcpy 窗口会打开，Android 画面已经在里面了。用 OBS 的窗口捕获对准它 —— 搞定。

3三个滑杆轻松调

  • Device — 来自 Android 手机的游戏声
  • Output — 你的 PC 声音（直播 BGM、Discord、浏览器）
  • Mic — 你自己的声音

在 ChargeCast 里把所有声音混好，交给 OBS 一路干净的输入。再也不用直播到一半跟混音器较劲。

从冷启动到这里大概 3 分钟。之后只要按 OBS 的「开始直播」就行。

常见问题

需要 root 或特殊的开发环境吗？

不需要。只要在 Android 的开发者选项里打开「USB 调试」即可。

能同时听到游戏声和自己的声音吗？

可以。三个通道各有独立滑杆。

应该搭配哪种直播软件？

OBS Studio 是推荐方案（使用「窗口捕获」源）。XSplit 等同类软件同理。

竖屏游戏怎么办？

方向可锁定为竖屏、横屏或自动，这样 OBS 布局就能保持稳定。

它会偷偷联网吗？

所有采集都通过本地 ADB。ChargeCast 本身不会把设备信息发到任何地方。

和 scrcpy 或 DroidCam 有什么不同？

ChargeCast 把 scrcpy 与 3 通道音频混音器以及一键 GUI 打包在一起，让游戏音和麦克风作为一路干净的来源进入 OBS。原版 scrcpy 不一定能传游戏声（仅 Android 11+ 且有限制），而 DroidCam 是把手机当作网络摄像头来用，并不针对屏幕镜像。ChargeCast 围绕直播主的实际流程设计：插 USB、边充电、OBS 窗口捕获。

关于 OBS 集成的说明

ChargeCast 本身不是直播软件。把它想成一个舞台，把干净的画面交给 OBS。

  • 让 OBS Studio 的窗口捕获 对准 ChargeCast 窗口 —— 这就是全部集成步骤
  • 画面、设备音和麦克风都已在 ChargeCast 端混好，所以 OBS 场景可以保持简洁
  • XSplit、Streamlabs、Discord 屏幕共享 —— 任何支持窗口捕获的工具都是同样用法

OBS 场景一旦配好，后续开播就一个动作：插上线，按 Start Capture。

Direct .exe install

The direct download is the same ChargeCast you get from the Microsoft Store, packaged as a standalone Inno Setup installer instead of an MSIX. It installs to %LOCALAPPDATA%\Programs\ChargeCast per-user (no admin prompt).

The installer is unsigned, so Windows SmartScreen warns the first time you run it:

  1. On the blue Windows protected your PC dialog, click More info
  2. Click Run anyway

To confirm the installer matches the official release, check the SHA256 hash listed on the Releases page: 

Get-FileHash -Algorithm SHA256 .\ChargeCast-1.7.0-Setup.exe
